#3 一些新的思考，即将到来的新体验

suriwang(王昕) · 2026-04-29 17:27 · 可见范围：SSV 全体员工
收到很多同事的体验反馈，如下是近期的思考，我们会这么做下去：
  1. 通知、触达、移动化：nexus 偏严肃场景，以 PC 为主，但移动端同样重要，企微 Bot + 移动端适配会分两期推进。
  2. AI 从单 topic 到交叉分析到纵深使用：单 topic 总结 → 交叉分析 → 放开 chat 输入框。
  3. 评论具备局限性，讨论能放大可能性：评论回复区会做一次升级改造，营造讨论氛围。

4 / 4 Thread 从发起到闭环的 6 步全流程。每步对应需求文档 §5。
全链路流程 以"红点消失逻辑讨论"为例
1
发起讨论
钟嘉辉在 Event 讨论区输入问题 → 点击 发起讨论 → 讨论区顶部新增 Thread 卡片（收起态）。
2
触发通知（新 Thread 发起）
系统向 Topic 订阅者/维护人推送 站内 Topic 动态 Tab + 企微 Bot 聚合推送。
3
多人参与（Thread 内线性回复）
王昕、李丹阳、李培轮点击通知进入 → 展开 Thread → 在 Thread 内回复。每次新回复自动通知 Thread 所有参与者。
4
产出结论 + @ 指派（高优通知）
肖迈总结讨论 → @钟嘉辉 → 被 @ 者收到 站内 @我 Tab + 企微 Bot（高优） 双通道通知。
5
📌 讨论转为待办（关键差异点）
肖迈点击消息 ··· 菜单 → 转为待办 → 指派钟嘉辉。Thread 卡片出现 ✅ 已转为待办；Event 待办列表新增一条并链回原 Thread。
✓
待办完成 → 状态同步回 Thread
钟嘉辉勾选完成 → Thread 底部系统消息 钟嘉辉 已完成关联待办 → 所有参与者收到通知。闭环。
